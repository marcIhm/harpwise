Overview of 'harpwise develop':

  This mode is useful only for the maintainer or developer of
  harpwise; it is therefore not listed in the general usage
  information.

  The required additional argument specifies a task to perform; it can be
  one of:

    - docs-make-man:
  [2m    process/produce man page from its own erb-template[0m
    - docs-diff-man:
  [2m    compare produced man page with usage[0m
    - docs-make-org-txt:
  [2m    process template from docs/erb-org and produce files[0m
  [2m    in _org and _txt[0m
    - docs-make-html:
  [2m    process org-files with org-publish from docs/_org and produce[0m
  [2m    file in _html[0m
    - docs-all:
  [2m    all docs-xyz extras one after the other[0m
    - selftest:
  [2m    perform selftest with special regard to the environment[0m
    - unittest:
  [2m    unittest for features, that cannot easily be tested within the[0m
  [2m    main test program[0m
    - widgets:
  [2m    excercise selected widgets to be tested[0m
    - lickfile:
  [2m    load and parse a lickfile[0m
    - check-frequencies:
  [2m    recalculate and check frequencies for current set of samples[0m
    - read-scale-with-notes:
  [2m    read given scale (first arg) from file (second arg)[0m
    - dump:
  [2m    dump internal data structures to stdout[0m
